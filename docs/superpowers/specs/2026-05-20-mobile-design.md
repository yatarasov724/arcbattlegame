# Мобильная версия Archaeology Battle

**Дата:** 2026-05-20  
**Подход:** CSS-first + минимальный JS (локальный стейт там, где нужен)

---

## 1. Фундамент (все экраны)

### Брейкпоинт
- `≤640px` — мобильный телефон (добавляется к существующим 768px и 480px)

### Размер клеток
- `--cell-size: 34px` при `≤640px` (сейчас 26px при ≤480px)
- `--cell-size-sm: 24px` при `≤640px` (не меняется)
- Горизонтальный padding экранов уменьшается до `8px` на `≤640px` (сейчас 16px)
- Расчёт: 10×34px + 9×1px (gap) + 16px (8px с каждой стороны) = 365px — помещается в iPhone SE (375px) без скролла
- `--grid-gap` при `≤640px` остаётся 1px (уже задан при `≤768px`)

### Touch UX
- `touch-action: manipulation` на всех интерактивных клетках сетки (`GridCell`, `PlacementGrid`) — убирает 300ms задержку на iOS
- Hover-эффекты остаются (на тач-устройствах они просто не срабатывают)

### Скрываемые элементы
- Подсказка `(R — переключить)` скрывается через `@media (max-width: 640px) { display: none }`

### Не меняется
- Визуальный стиль: цвета, шрифты, CSS-переменные — без изменений
- `viewport` мета уже стоит в `index.html`
- Никаких новых зависимостей

## 2. Экран битвы (`BattleScreen`)

### Проблема
На `≤640px` две сетки 10×10 рядом не помещаются (нужно ~800px).

### Решение: вкладки

**Новое состояние в `BattleScreen.tsx`:**
```ts
const [activeTab, setActiveTab] = useState<'human' | 'ai'>('ai');
```

**Таб-бар** — две кнопки между HUD и сеткой, видны только на `≤640px`:
- `📍 Моё поле` → показывает свою сетку
- `⚔️ Противник` → показывает сетку врага

**CSS:** обе `.gridSection` рендерятся всегда, неактивная скрыта через `display: none` на мобильном. На `≤640px` `.main` переключается в `flex-direction: column`. Центральная панель получает `order: 10` — всегда оказывается последней, под активной сеткой. Скрытая `.gridSection` не влияет на поток благодаря `display: none`.

**Автопереключение (в `useEffect` при изменении `state.turn` и `state.actionMode`):**
- `turn === 'human'` и `actionMode === 'dig'` → переключить на `'ai'`
- `turn === 'ai'` → переключить на `'human'`
- `actionMode === 'move'` (режим перемещения) → переключить на `'human'`

Пользователь может переключать вкладки вручную в любой момент — автопереключение только при смене хода/режима.

### Центральная панель на мобильном
На `≤640px` центральная панель (кнопки действий + дневник раскопок + спиннер ИИ) показывается под сеткой в горизонтальном `flex-wrap` режиме. Дневник (`log`) ограничен `max-height: 80px`.

## 3. Экран расстановки (`PlacementScreen`)

### Проблема
На `≤640px` сетка + боковая палитра рядом не помещаются.

### Решение: нижняя панель артефактов

На `≤640px` `ArtifactPalette` меняет раскладку:
- Вместо вертикального списка карточек — горизонтальная сетка 4 иконки
- Каждая иконка: эмодзи (20px) + счётчик оставшихся (`×N` / `✓ готово`)
- Панель прикреплена снизу внутри `.main`, занимает полную ширину

**Кнопка ориентации:**
- На `≤640px` строка `orientationRow` (горизонталь/вертикаль/подсказка) заменяется компактной кнопкой-переключателем (`↔` / `↕`) в правом верхнем углу над сеткой
- Реализуется через CSS: скрыть `orientationRow` на мобильном, показать отдельный элемент `orientationBtnMobile` (рендерится всегда, скрыт на десктопе)

**Кнопки действий** (`Случайно`, `Очистить`, `Начать раскопки`) — на `≤640px` растягиваются на всю ширину (`width: 100%`).
